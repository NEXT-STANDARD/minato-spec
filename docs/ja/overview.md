# MINATO 概要

> 5分で読める MINATO の紹介
> 詳しい仕様は [`MINATO_PROTOCOL.md`](../../MINATO_PROTOCOL.md) を参照してください。

---

## ひとことで言うと

MINATO（みなと、湊）は、**あなたのスマホと友達のスマホのAIエージェント同士が直接会話するための共通ルール**です。

アプリではありません。プロトコル——つまり「話し方の約束ごと」です。
インターネットが TCP/IP という約束ごとの上に成り立っているように、
将来のエージェント同士のやりとりも、何かしらの共通ルールが必要です。
MINATO はその候補のひとつとして設計されています。

---

## なぜ必要なのか

### シーン 1: 飲み会の日程調整

```
あなた「来週飲みに行こうよ！」
友人  「いいね、いつにする？」
あなた「月曜は予定入ってて…火曜は会議…木曜なら空いてる」
友人  「木曜いいね。でも何時？」
あなた「19時？あ、でもその日…」
```

このやりとり、2人のエージェントに任せられたらどうでしょう。

```
あなたのエージェント ──► 友人のエージェント
「木曜の19時-21時が空いています。候補の場所は3つあります」
                    ◄──
「木曜19時、承諾しました。渋谷の〇〇でどうですか？」

あなた「木曜19時、渋谷の〇〇でどう？」 ← スマホが提案してくる
```

人間は最後の一言だけ確認すればいい。
面倒な調整は、寝ている間にエージェント同士が終わらせています。

### シーン 2: 言葉が通じない人との会話

```
日本人   「カレーの辛さってどれくらい？」
アメリカ人「?????」
```

MINATO の世界では、こうなります:

```
日本人のエージェント ──► アメリカ人のエージェント
  content: "カレーの辛さってどれくらい？"
  translated_content: "How spicy is this curry?"

アメリカ人のエージェント ──► 日本人のエージェント
  content: "Medium spicy, you'll be fine"
  translated_content: "中辛くらいだよ、大丈夫"
```

両者はそれぞれ自分の母語で話しているだけ。
**翻訳はエージェント同士が裏でやっていて、プロトコルの中に組み込まれています。**

## 4つの特徴

### 1. 誰のものでもない

MINATO は特定の会社の製品ではありません。MITライセンスのオープンな仕様書で、
誰でも自由に実装できます。Apple も Google もあなたの友達も、同じルールで話せます。

### 2. インターネットがなくても動く

近くにいる相手とは、**Bluetooth (BLE Mesh)** で直接つながります。
電波がなくても、Wi-Fiがなくても、エージェント同士は話せます。
遠くの相手とは、**Nostr**（分散型メッセージネットワーク）経由で話します。
この切り替えは自動です。

### 3. どこまで任せるかを選べる

相手ごとに、エージェントの自律度を 4 段階で設定できます:

| モード | 挙動 |
|--------|------|
| 見習い | 提案するだけ。実行は全部あなたが決める |
| 相棒 | 実行する直前に一度だけ確認する |
| 右腕 | 小さなことは自動、大事なことだけ聞く |
| 分身 | 全部任せる。事後通知のみ |

初対面の人には「見習い」、親友には「分身」、
というように使い分けることを想定しています。
これは Claude Code の permission mode から着想を得た設計です。

### 4. どの AI でも繋がる

Claude でも GPT でも Gemini でも、どの AI エンジンを使っているかは
プロトコルレベルでは関係ありません。エージェントが MINATO の話し方を
知っていれば、中身の頭脳は何でもいいのです。

## 名前の由来

**湊（みなと）** は、人・物・金・情報が集まる交通のハブです。
船が出発し、到着する場所。
MINATO は、エージェントが集まるそんなハブを目指しています。

> エージェントが間に入ることで、言語はただのデータ形式になる。
> 人間は母語で話すだけでいい。
